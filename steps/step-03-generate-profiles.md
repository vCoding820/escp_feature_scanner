# Step 3: 生成 YAML 画像

## MANDATORY EXECUTION RULES:

- 🛑 NEVER deviate from the fixed YAML schema
- ✅ ALWAYS use knowledge-profile-schema.yaml as the schema template
- 📋 Generate exactly 21 YAML files — one per feature
- 💾 Save all profiles to `{output_folder}/features/`
- 🔄 未检测到的特性也要生成画像（字段留默认空值）

## CONTEXT BOUNDARIES:

- Feature scan results from step-02
- Fixed schema from `./templates/knowledge-profile-schema.yaml`

## YOUR TASK:

为全部 21 个特性生成标准化 YAML 画像文件，严格遵循固定 schema。

## EXECUTION:

### 1. 加载 Schema 模板

读取：
- `./templates/knowledge-profile-schema.yaml` — 固定 YAML schema

保存为 `{output_folder}/_index.yaml`：

```yaml

project: "[project_path]"
total_features: 21
detected_features: N
suspected_features: M
not_detected_features: K

categories:
  通信总线: [can, uart, iic, eth]
  通信: [plc]
  无线通信: [wifi, bluetooth, nearlink, 4g]
  存储: [flash]
  外设: [adc, dido, lcd, led]
  采集: [bmic]
  系统软件: [bios, bootloader, rtos, linux, keep]
  多核: [multicore]

  - id: can
    name: CAN总线通信
    status: detected      # detected / suspected / not_detected
    files_count: 12
    loc: 450
  # ... 全部21个特性
```
